## Firestore 트래픽 최적화 계획

감사 결과 5개 핵심 문제를 발견했습니다. 모두 `increment()` 원자 연산 + 메타 문서 캐시 패턴으로 해결합니다.

### 최적화 1: QuizPage — 답변당 getDoc 제거 (10문제 = 11 reads → 1 read)

**현재**: 매 답변마다 `getDoc(studentProgress)` → `setDoc` (read-modify-write)
**개선**: 
- 세션 시작 시 studentProgress 1회만 getDoc → ref에 캐싱
- 답변 시 `setDoc({ merge: true })` + `increment()`만 사용 (읽기 없음)
  ```
  total: increment(1)
  correct: increment(isCorrect ? 1 : 0)
  wrong: increment(isCorrect ? 0 : 1)
  totalXP: increment(isCorrect ? 10 : 0)
  todayCount: 오늘 첫 풀이면 1로 set, 아니면 increment(1)
  ```
- streak/todayCount 로직은 세션 시작 시 읽은 캐시로 판단 (하루에 한 번만 갱신)
- `endSession()`도 캐시 ref 사용 (getDoc 제거)
- **절감: 10문제 세션당 11 reads → 1 read**

### 최적화 2: StudentDashboard — studentAttempts 전체 스캔 제거

**현재**: 대시보드 마운트마다 학생의 모든 attempt 문서 읽음 (무한 증가)
**개선**:
- studentProgress 문서에 `topicStats` 맵 추가: `{ Algebra: { total: 5, wrong: 2 }, ... }`
- handleAnswer에서 increment로 토픽별 통계 갱신: `topicStats.${topic}.total: increment(1)`
- Dashboard는 studentProgress 1문서만 읽어서 추천 생성
- **절감: N docs (증가함) → 1 doc**

### 최적화 3: AdminDashboard — 3개 풀스캔 + 중복 리스너 제거

**현재**: 
- `meta/pendingStudents` onSnapshot 중복 (AuthContext에 이미 있음)
- users + students + questions 3개 컬렉션 풀스캔 (매 version 변경마다 재실행)

**개선**:
- 중복 onSnapshot 제거 → AuthContext의 `pendingVersion`을 useEffect 의존성으로 사용
- 메타 문서 캐시 패턴 도입: `meta/appStats` 단일 문서
  ```
  { version, userCount, studentCount, questionCount, updatedAt }
  ```
- AdminDashboard는 `meta/appStats` 1문서 onSnapshot → count만 표시
- users 목록은 `pendingVersion` 변경 시에만 새로고침
- 모든 mutation 시 `meta/appStats.version` increment
- **절감: 3 full scans → 1 doc read**

### 최적화 4: AdminDashboard users 목록 — 버전 기반 캐싱

**현재**: users 전체를 매번 새로 읽음
**개선**: 
- pendingVersion 변경 시에만 users 새로고침 (이미 AuthContext에 있음)
- localStorage에 users 캐싱 (버전 기반 무효화)

### 변경 파일
1. `src/pages/QuizPage.jsx` — increment 기반 답안 저장, 세션 시작 시 1회만 getDoc, topicStats 갱신
2. `src/pages/StudentDashboard.jsx` — studentProgress 1문서만 읽기, topicStats에서 추천 생성
3. `src/pages/AdminDashboard.jsx` — 중복 리스너 제거, meta/appStats 카운터 사용, pendingVersion 기반 새로고침
4. `src/pages/StudentsPage.jsx` — approve 시 meta/appStats version bump
5. `firestore.rules` — meta/appStats 규칙 (이미 meta/{docId}로 커버됨)

| 시나리오 | Before | After |
|----------|--------|-------|

| 학생 대시보드 로드 | 1 + N attempts | 1 read |
| Admin 대시보드 로드 | 3 full scans + 중복 리스너 | 1 doc |
